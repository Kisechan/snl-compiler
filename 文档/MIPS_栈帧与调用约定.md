# MIPS 栈帧与调用约定

本文记录当前 `snlc --mips` 后端使用的 MARS 兼容 32 位 MIPS 栈帧和过程调用规则。

## 栈帧布局

调用者在 `jal` 前一次性分配调用区：

```text
0($sp)                    第 1 个参数
...                       后续参数
argument_size($sp)        static link
```

被调过程进入后建立自己的栈帧：

```text
0($fp)                    saved $ra
4($fp)                    saved old $fp
8($fp)                    第 1 个参数
...                       后续参数
8 + argument_size($fp)    static link
-4($fp), -8($fp), ...     局部变量区
```

全局变量仍分配在 `.data` 段中。过程的局部变量和参数通过 `$fp` 相对地址访问。

## 参数传递

- 值参数：标量参数传值；数组和记录值参数按字节复制到调用区。
- `var` 参数：传递实参左值地址；形参访问时先从参数槽取出目标地址。
- 嵌套过程调用时，调用者根据被调过程的词法父作用域计算 static link。
- 递归调用自然建立新的栈帧，因此局部变量和参数互不覆盖。

## 变量寻址

后端统一先生成左值地址，再执行读写：

- 全局变量：使用 `.data` 标签地址。
- 当前过程局部变量：使用 `$fp` 负偏移。
- 参数：使用 `$fp` 正偏移；`var` 参数额外解引用一次。
- 外层过程变量：沿 static link 找到声明层级的栈帧，再按变量偏移寻址。
- 数组元素：`base + (index - low) * element_size`。
- 记录字段：`base + field.offset`。

## 当前覆盖范围

当前后端覆盖现有语法和语义阶段支持的主要 SNL 子集：

- 全局变量、局部变量、值参数、`var` 参数。
- `integer`、`char`、数组、记录。
- 过程声明、嵌套过程、过程调用和递归。
- 赋值、`read`、`write`、`if`、`while`、`return(expr)`。
- 整数算术、`<` 和 `=` 标量比较。

SNL 过程没有返回类型；`return(expr)` 会计算表达式，然后提前跳转到当前过程出口。
